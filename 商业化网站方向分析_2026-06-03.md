# AI PickGold 商业化网站方向分析

日期：2026-06-03
域名：https://www.aipickgold.com
主题：自媒体运营、个人 IP、知识博主、轻创业网站方向

## 一句话结论

最值得做的不是“个人主页”、不是“纯聊天机器人”，也不是只卖“排版工具”。

最有商业价值的方向是：

> 把 aipickgold.com 做成 AI PickGold Studio：一个面向知识博主和 AI 创作者的内容生产工作台。它把一个选题/Markdown 草稿，转换成公众号文章、小红书卡片、X 帖子、短视频脚本、发布包和 API/Skill 输出，同时保留人工审核与手动发布边界。

你的优势不在于“又做一个 AI 工具站”，而在于你已经有真实运营流程：

- WxMD：公众号 Markdown 排版。
- CardMD：小红书知识卡片。
- 自媒体运营库：选题、平台改写、风险审查、人工发布。
- HyperFrames：视频即代码、短视频模板、课程/案例潜力。
- Codex/GitHub/Product Design/Linear/Figma：可以变成你“用 AI 做产品和内容”的公开案例。

## 关键判断

1. 内容本身越来越卷，工具化的运营流程更值钱。
2. 聊天机器人可以做，但必须建立在你的案例库、模板库、发布规则之上，不能单独作为主产品。
3. 视频方向很有商业潜力，但不要做泛泛的 AI 视频生成器，应该做“知识视频工作流/模板/课程”。
4. 排版 API 是你目前最接近收入的资产，应该升级为 Skill/API Center。
5. 个人 IP 的商业重心应是“把 AI 工作流拆成可执行、可审查、可交付、可变现的系统”。

## 方向 1：AI PickGold Studio 工作台

### 网站形态

第一屏就应该是工具，不要做空泛 landing page。

推荐结构：

- 左侧：原始 Markdown / source unit。
- 中间：平台输出切换。
- 右侧：预览和发布包状态。
- 顶部标签：公众号、小红书、X、视频、API。
- 底部状态：待审查、可手动发布、阻塞、已导出。

### 核心卖点

用户不是为“好看的主题”付费，而是为节省从“草稿”到“可发布包”的时间付费。

可以卖：

- 一份选题生成多平台版本。
- 公众号排版。
- 小红书卡片。
- X thread。
- 短视频脚本。
- 发布清单。
- 风险审查。
- 手动发布交接包。

### 变现方式

- Free：有限主题/有限导出/带水印。
- Pro：无限导出、去水印、更多模板。
- Studio：多平台发布包、历史记录、品牌模板、API key。
- Custom Setup：帮创作者搭建自己的内容工作流。

建议早期价格：

- Pro：29-49 元/月。
- Studio：99-199 元/月。
- 早鸟终身版：399-899 元。
- 私人工作流搭建：999-2999 元起。

## 方向 2：Skill/API Center

这是你目前最接近商业化的部分。

### 可以做的 Skill

- WxMD Skill：把 Markdown 文章变成公众号 HTML。
- CardMD Skill：把教程/观点变成 5-7 页小红书卡片。
- Creator Package Skill：把一个选题变成公众号、小红书、X、短视频版本。
- Review Skill：检查平台风险、AI 标识、标题夸张、CTA 和发布边界。
- HyperFrames Video Skill：把案例变成短视频脚本、镜头表、字幕和渲染说明。

### 可以做的 API

- Markdown -> 公众号 HTML。
- Markdown/结构化 JSON -> 小红书卡片。
- Source unit -> 多平台发布包 manifest。
- Theme preview / theme validation。

### 不建议第一阶段主打

不要把“一键自动发布全平台”作为卖点。平台风险高，和你当前人工审核边界也冲突。

### 变现方式

- API 月度额度。
- API 点数包。
- 团队版。
- 私有部署。
- 定制 Skill 服务。

## 方向 3：案例库 + 模板商店

这是信任层，也是最快能收钱的层。

### 每个案例应该展示

- 原始选题。
- 用户痛点。
- 输入草稿。
- 使用的 prompt / skill / workflow。
- 公众号输出。
- 小红书卡片输出。
- X 输出。
- 短视频脚本。
- 风险审查。
- 最终人工发布说明。
- 可复用模板。

### 可以卖的模板

- Source unit 模板。
- 小红书 7 卡模板。
- 公众号文章结构模板。
- X thread 模板。
- 短视频分镜模板。
- 平台风险审查表。
- 发布包 manifest 模板。

### 为什么要立刻做

知识工具最大的阻力是用户不相信“这个流程真的能产出东西”。案例库能解决这个问题。

## 方向 4：HyperFrames Video Lab / Creator Kit

这个方向适合做你的第二主线。

### 定位

不要说“AI 一键生成视频”，更好的说法是：

> 用 AI 生成镜头资产，用 HyperFrames 做导演台和剪辑台，用真人判断保留内容可信度。

### 网站内容

- 视频工作台。
- 短视频脚本生成。
- 分镜/字幕/封面/音效清单。
- HyperFrames 模板。
- 真实案例拆解。
- 课程或 Creator Kit。

### 变现方式

- 模板包：99-299 元。
- 小课/训练营：199-999 元。
- 私人视频工作流搭建：1999-9999 元。
- 与 Studio 订阅打包。

## 方向 5：个人 IP 知识机器人

可以做，但不应该最先做。

### 正确做法

机器人应该回答你的公开工作流和案例，而不是泛泛聊天：

- 这个选题适合做小红书还是公众号？
- 如何把一篇文章拆成 7 张卡？
- 这个标题有没有平台风险？
- 这个 Markdown 怎么变成公众号格式？
- 这个视频脚本怎么拆成 HyperFrames 镜头？

### 数据来源

第一阶段只用公开材料：

- 网站文档。
- 公开案例。
- 模板说明。
- 公开 changelog。
- 公开个人 IP playbook。

不要把私密备忘录、token、未公开财务、账号细节、客户资料喂给公开 bot。

### 什么时候做

等案例库有 10-20 个真实案例后再做。那时候 bot 会像销售顾问和工作流助手，而不是套壳聊天框。

## 方向 6：AI 工具 / GitHub Skill 雷达

这个方向适合当内容引流，不适合做第一商业主线。

可以做：

- 每周 AI 工具观察。
- GitHub 项目拆解。
- Codex Skill 安装案例。
- “这个工具能不能变成创作者工作流”的评分。

风险是容易变成普通 AI 资讯站，所以必须绑定你的工作流、模板和实际案例。

## 方向 7：Tushare / 证券研究网站

这条线暂时不要混进 AI PickGold 主站。

原因：

- 受众不同。
- 合规风险更高。
- 容易让“创作者工具”变成“投资建议”误解。
- 你的主 IP 边界是不荐股、不承诺收益。

建议：证券研究继续保留为独立本地/Obsidian 研究系统，未来如果产品化，单独做子品牌。

## 推荐网站结构

优先级从高到低：

1. `/studio`
   - 多平台内容工作台。

2. `/skills`
   - WxMD、CardMD、Creator Package、Review、HyperFrames Video Skill。

3. `/api`
   - Convert API、API playground、key/quota/log。

4. `/cases`
   - 真实案例库。

5. `/templates`
   - 模板商店。

6. `/video-lab`
   - HyperFrames 视频工作流和课程/Creator Kit。

7. `/pricing`
   - Free、Pro、Studio、API、Custom Setup。

8. `/about`
   - 个人 IP、边界、真实运营故事。

## 30 / 60 / 90 天路线

### 1-30 天

- 把首页定位改成 AI PickGold Studio。
- 加 source unit / 多平台输出 / 发布包状态。
- 加 Skill Center 页面。
- 加 1 个真实案例。
- 对齐隐私、API、发布边界文案。
- 发布 3 篇小红书、3 条 X、1 篇公众号、1 个短视频，全部围绕“一个选题如何变成多平台发布包”。
- 开 3 个私人工作流搭建名额测试收费。

### 31-60 天

- 做 API Playground。
- 做额度/日志视图。
- 模板商店 v1。
- 案例库累积 5-10 个。
- 上线 HyperFrames Video Lab 入口。
- 测试 Pro、Studio、API 点数包、Creator Kit 等价格。

### 61-90 天

- 上线公开文档知识机器人。
- 支持保存品牌风格。
- 上线团队/定制服务页。
- 案例库累积 20 个以上。
- 做课程或 Creator Kit 预售。

- 公众号版本

- X 帖子/thread
- 短视频脚本
- 风险清单
- 发布包 manifest

包含：

- 问题
- 输入
- 流程
- 输出

## 最终建议

按这个顺序推进：

1. AI PickGold Studio 工作台。
2. Skill/API Center。
3. 案例库 + 模板商店。
4. HyperFrames Video Lab / Creator Kit。
5. 公开文档知识机器人。

你的个人 IP 最适合定成：

> 我不是展示 AI 有多神奇，而是把 AI 工作流拆成能复用、能审查、能交付、能变现的系统。

这个定位能同时覆盖你的自媒体运营、知识博主、轻创业、排版 API、视频 Skill 和机器人方向。
